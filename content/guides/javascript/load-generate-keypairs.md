can generate and load private keys in the most common formats with Javascript.

## Different types of key material

Key material on Solana can be in virtually any format. The most common formats
are:

- Mnemonic/seed/secret phrase
- Byte array
- Base58 string
- Hexadecimal string

### Mnemonic/seed/secret phrase

A mnemonic is a string of human readable words that can be used to generate
multiple keypairs. These are commonly generated by wallets and are 12, 18, or 24
words long. The `solana-keygen` CLI tool also generates mnemonics by default.

They look like this:

```text
cabbage together debris chalk hammer novel wire tunnel rubber labor retire prefer
```

These are the only type of key material that can be used to generate multiple
private keys.

### Byte array

A byte array is a raw binary representation of a key in the form of a list of
byte values. This is the default format used by the Solana web3.js library.

```text
[248,212,171,183,118,56,150,86,198,206,58,187,102,63,92,131,18,24,151,162,185,185,42,19,139,86,99,186,160,177,149,229,207,151,106,210,8,20,167,108,215,120,87,203,3,8,224,142,218,214,122,119,109,109,113,73,91,238,214,54,60,109,206,69]
```

### Base58 string

A text representation of the key in Base58 format. The Base58 format excludes
characters that are easily confused with each other, such as `0` and `O`, or `I`
and `l`. This makes it easier to manually copy and since there are no characters
that are typically used as delimiters, it's compatible with double-click
selection. It's also much more compact than a byte array or hex string.

```text
5yYeHuwyyy4qAqJtvkkA2SDsujQorZD8SvJ3v6YevYEpaVtq5M2vSpSr3Ts8Mzm2gAwKtPqvozpxLejUxgkbpa5a
```

### Hex string

A text representation of the key in hexadecimal format. Each byte of the key is
represented by two hexadecimal characters. This is not used often, but it's here
for completeness.

```text
f8d4abb776389656c6ce3abb663f5c83121897a2b9b92a138b5663baa0b195e5cf976ad20814a76cd77857cb0308e08edad67a776d6d71495beed6363c6dce45
```

## Generate keypairs

### Setup

To get started, create a new directory, initialize a new npm project, and
install dependencies:

```bash
mkdir solana-keypairs
cd solana-keypairs
npm init -y
npm install @solana/web3.js bip39 bs58 ed25519-hd-key buffer tweetnacl dotenv
```

### Generate a keypair from a mnemonic

Next, create a new file called `index.js` and add the following code:

```javascript
import { Keypair } from "@solana/web3.js";
import * as bip39 from "bip39";
import bs58 from "bs58";
import fs from "fs";

// Generate a new mnemonic (seed phrase)
const mnemonic = bip39.generateMnemonic();
console.log("Mnemonic (Seed Phrase):", mnemonic);

// Convert the mnemonic to a seed
const seed = await bip39.mnemonicToSeed(mnemonic);
const seedBytes = seed.slice(0, 32); // Use the first 32 bytes
const keypair = Keypair.fromSeed(seedBytes);

// Extracting the secret key (default format is byte array)
const secretKey = keypair.secretKey;
console.log("Secret key as byte array:", secretKey);
```

This code will generate a new mnemonic, convert it to a seed, and then generate
a keypair from the seed. Printing the private key of this keypair will give us
the byte array representation of the key.

For reference, this is the default way of generating a keypair with web3.js:

```javascript
import { Keypair } from "@solana/web3.js";

const keypair = Keypair.generate();
console.log("Secret key as byte array:", keypair.secretKey);
```

### Convert the keypair to base58 and hex

Once we have a keypair, we can convert it to a base58 string or hex string.

Add the following code to the end of `index.js`:

```javascript
// Converting the private key to a base58 string
const base58PrivateKey = bs58.encode(keypair.secretKey);
console.log("Secret key in base58: " + base58PrivateKey);

// Convert the secret key to a hex string
const secretKeyHex = Array.from(secretKey)
  .map(byte => byte.toString(16).padStart(2, "0"))
  .join("");
console.log("Secret key in hex:", secretKeyHex);
```

Now, run the script:

```bash
node index.js
```

You should see output similar to this:


## Load keypairs of different types

Private keys are often loaded into a program from environment variables or
configuration files. Here's how you can load all the different types of key
material we've generated.

We'll continue using the same Node.js project from the previous section. Create
a new file called `.env` and add the following:

```text
SECRET_PHRASE=cabbage together debris chalk hammer novel wire tunnel rubber labor retire prefer
BYTE_ARRAY=[248,212,171,183,118,56,150,86,198,206,58,187,102,63,92,131,18,24,151,162,185,185,42,19,139,86,99,186,160,177,149,229,207,151,106,210,8,20,167,108,215,120,87,203,3,8,224,142,218,214,122,119,109,109,113,73,91,238,214,54,60,109,206,69]
BASE58_STRING=5yYeHuwyyy4qAqJtvkkA2SDsujQorZD8SvJ3v6YevYEpaVtq5M2vSpSr3Ts8Mzm2gAwKtPqvozpxLejUxgkbpa5a
HEX_STRING=f8d4abb776389656c6ce3abb663f5c83121897a2b9b92a138b5663baa0b195e5cf976ad20814a76cd77857cb0308e08edad67a776d6d71495beed6363c6dce45
```

### Load a keypair from a mnemonic

Create a new file called `load.js` and add the following code:

```javascript
import { Connection, Keypair, clusterApiUrl } from "@solana/web3.js";
import * as bip39 from "bip39";
import nacl from "tweetnacl";
import { Buffer } from "buffer";
import * as ed25519 from "ed25519-hd-key";
import bs58 from "bs58";
import dotenv from "dotenv";
dotenv.config();

// Load from mnemonic
const mnemonic = process.env.SECRET_PHRASE;
const seed = bip39.mnemonicToSeedSync(mnemonic).slice(0, 32);

// You can create a keypair directly from the mnemonic by converting it into a seed
const seedPhraseKeypair = Keypair.fromSeed(seed);
console.log(
  "Public key from seedPhraseKeypair:",
  seedPhraseKeypair.publicKey.toBase58(),
);
```

Optionally, you may want to create a keypair using a custom derivation path.
Here's how this can be done:

```javascript
// To create a keypair using a custom derivation path
// Define the derivation path
const path = `m/44'/501'/0'/0'`; // Replace with your custom path
// Derive a seed from the given path
const derivedSeed = ed25519.derivePath(path, Buffer.from(seed, "hex")).key;
// Generate a keypair from the derived seed using tweetnacl (NaCl = Networking and Cryptography library)
const derivedUint8Keypair = nacl.sign.keyPair.fromSeed(derivedSeed);
// This is a Uint8Array, not a Solana web3.js Keypair object, so you will need to convert it
const customPathKeypair = Keypair.fromSecretKey(
  Uint8Array.from(derivedUint8Keypair.secretKey),
);
```

### Load a keypair from a byte array, base58 string, or hex string

Creating keypairs from byte arrays, base58 strings, and hex strings is
straightforward these can be decoded and converted to Uint8Arrays using the
`bs58` and `tweetnacl` libraries.

Add to the end of `load.js`:

```javascript
// Load from Byte Array
const byteArray = JSON.parse(process.env.BYTE_ARRAY);
const byteArraySecretKey = Uint8Array.from(byteArray);
const byteArrayKeypair = Keypair.fromSecretKey(byteArraySecretKey);
console.log(
  "Public key from byteArrayKeypair:",
  byteArrayKeypair.publicKey.toBase58(),
);

// Load from base58 encoded string
const base58String = process.env.BASE58_STRING;
const base58SecretKey = bs58.decode(base58String);
const base58Keypair = Keypair.fromSecretKey(base58SecretKey);
console.log(
  "Public key from base58Keypair:",
  base58Keypair.publicKey.toBase58(),
);

// Load from hexadecimal string
const hexString = process.env.HEX_STRING;
const hexStringSecretKey = new Uint8Array(
  hexString.match(/.{1,2}/g).map(byte => parseInt(byte, 16)),
);
const keypairFromHex = Keypair.fromSecretKey(hexStringSecretKey);
console.log(
  "Public key from keypairFromHex:",
  keypairFromHex.publicKey.toBase58(),
);
```

Running this script with

```bash
node load.js
```

will give you output similar to this:

```text

Public key from base58Keypair: EyMJqtMi9mvFpBatswmm4VWnA4WvAX5jRzDfLVkyfzig
Public key from keypairFromHex: EyMJqtMi9mvFpBatswmm4VWnA4WvAX5jRzDfLVkyfzig
```
