订阅以在新区块为`confirmed(已确认)`或`finalized(已完成)`时接收通知。


<Callout type={"caution"} title={"不稳定的方法"}>
  此订阅被认为不稳定，并且仅在使用该`--rpc-pubsub-enable-block-subscription`标志启动验证器时才可用。
  此订阅的格式将来可能会发生变化。
</Callout>

<DocSideBySide>

<DocLeftSide>

### 参数

<Parameter name="过滤器" type={"string | object"} required={true}>

  日志的过滤条件以按帐户类型接收结果；目前支持：

  <Field type="string" value="all">
  `all` - 包含区块中的所有交易
</Field>

<Field type="object">

  具有以下字段的 JSON 对象：
  - `mentionsAccountOrProgram: <string>` - 仅返回提及所提供的公钥（作为 base-58 编码字符串）的交易。如果给定块中没有提及，则不会发送任何通知。

</Field>

</Parameter>

<Parameter type={"object"} optional={true}>

  配置对象包含以下字段：

  <Field
  name="承诺"
  type="string"
  optional={true}
  defaultValue={"finalized"}
  href="/docs/rpc/index.zh.mdx#配置状态承诺"
>

- `processed` 不支持。

</Field>

<Field 
  name="编码"
  type="string"
  defaultValue="json"
  optional={true}
  href="/docs/rpc/index.zh.mdx#解析后的响应"
>

  每个返回的Transaction的编码格式

  <Values values={["json", "jsonParsed", "base58", "base64"]} />

<details>

- `jsonParsed` 尝试使用特定于程序的指令解析器在列表中返回更多人类可读和明确的数据 `transaction.message.instructions`。
- 如果 `jsonParsed` 请求但找不到解析器，则指令将回退到常规 JSON 编码（`accounts`、`data`和`programIdIndex` 字段）。

</details>

</Field>

<Field
  name="交易详情"
  type="string"
  optional={true}
  defaultValue={"full"}
>

  要返回的交易详细信息级别

  <Values values={["full", "accounts", "signatures", "none"]} />

<details>

- 如果 `accounts` 有要求，交易详细信息仅包括每笔交易中的签名和带注释的帐户列表。
- 交易元数据仅限于: fee, err, pre_balances,
  post_balances, pre_token_balances, 和 post_token_balances。

</details>

</Field>

<Field name="最大支持的交易版本" type="number" optional={true}>

  在响应中返回的最大交易版本。

  <details>

- 如果请求的区块包含更高版本的交易，则会返回错误。
- 如果省略该参数，则仅返回遗留交易，并且包含任何版本化交易的块将提示错误。

</details>

</Field>

<Field name="显示奖励" type="bool" optional={true}>
  是否填充`rewards`数组。如果不提供参数，则默认包含奖励。
</Field>

</Parameter>

### 结果

`integer` - 订阅 ID（取消订阅时需要）

</DocLeftSide>

<DocRightSide>

### 代码示例

```json
{
  "jsonrpc": "2.0",
  "id": "1",
  "method": "blockSubscribe",
  "params": ["all"]
}
```

```json
{
  "jsonrpc": "2.0",
  "id": "1",
  "method": "blockSubscribe",
  "params": [
    {
      "mentionsAccountOrProgram": "LieKvPRE8XeX3Y2xVNHjKlpAScD12lYySBVQ4HqoJ5op"
    },
    {
      "commitment": "confirmed",
      "encoding": "base64",
      "showRewards": true,
      "transactionDetails": "full"
    }
  ]
}
```

### 响应

```json
{ "jsonrpc": "2.0", "result": 0, "id": 1 }
```

</DocRightSide>
</DocSideBySide>
